## 🤖 AI Collaboration During Development

### How AI Was Used
Claude Code (VS Code) was used throughout this project as a coding assistant. It was used to scaffold new files, implement functions from detailed specifications, fix syntax errors, and generate the evaluation harness structure.

### One Helpful AI Suggestion
When asked to implement the `build_pet_context()` function, Claude Code produced a clean, readable multi-line string builder that correctly iterated through nested Owner → Pet → Task relationships. This saved significant time and matched the intended format exactly on the first attempt.

### One Flawed AI Suggestion
When asked to add `import os` to `ai_advisor.py`, Claude Code inserted it after the `from __future__ import annotations` line, which caused a `SyntaxError` because `__future__` imports must always be first. The fix required a follow-up prompt to reorder the imports correctly. This was a good reminder that AI-generated code always needs human review before running.

## ⚠️ System Limitations

- **No memory between sessions** — The AI does not remember previous conversations. Each query is independent.
- **Self-reported confidence** — The confidence score is generated by the model itself, not calculated from ground truth. It reflects the model's own estimate, which may not always be accurate.
- **Keyword-based guardrails** — The input filter uses a fixed blocklist. Sophisticated harmful queries that avoid blocked keywords could still pass through.
- **Pet data dependency** — Response quality depends entirely on how much pet data the user has entered. An owner with no tasks will receive generic advice.
- **No veterinary authority** — The system explicitly does not replace professional veterinary advice. It is an organizational and informational tool only.

---

## 🔒 Potential Misuse & Mitigations

| Risk | Mitigation |
|------|-----------|
| User asks for harmful pet treatment advice | Blocked keyword list prevents obvious harmful queries |
| User tries to use the advisor for non-pet topics | System prompt instructs the model to redirect off-topic questions |
| Low-quality or hallucinated responses shown to user | Output validator checks response length and flags known failure phrases |
| Undetected bad responses | All interactions logged to `advisor_log.jsonl` for audit |

## 😮 What Surprised Me During Testing

The guardrail system worked more reliably than expected — blocked keyword detection was instant and never produced false negatives for the tested cases. What was surprising was how gracefully the system handled API failures: rather than crashing the entire app, it returned a structured error dict that the UI displayed cleanly. This made the error handling feel like a feature rather than a fallback.
